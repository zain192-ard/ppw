Web Crawling#

—#

Markdown cell untuk deskripsi#

“”” Web crawling, atau sering disebut “web spider” atau “bot,” adalah proses otomatis yang dilakukan oleh program komputer untuk menelusuri World Wide Web dan mengumpulkan informasi dari halaman web secara sistematis.

Secara sederhana, web crawler bekerja seperti pustakawan digital yang terus-menerus membaca dan mengumpulkan data dari miliaran halaman web yang ada di internet. Tujuan utamanya adalah untuk membuat indeks dari konten-konten tersebut agar nantinya dapat digunakan oleh mesin pencari, seperti Google, untuk menampilkan hasil yang relevan ketika pengguna melakukan pencarian. “””

—#

Markdown cell untuk sub-judul#

Cara Kerja Web Crawling#

Markdown cell untuk penjelasan bertahap#

“”” Proses web crawling umumnya meliputi tahapan berikut:

  1. Mengunjungi URL Awal: Crawler memulai dari sebuah URL (Uniform Resource Locator) atau daftar URL yang sudah diketahui.

  2. Membaca Konten: Setelah sampai di sebuah halaman, crawler akan membaca seluruh konten yang ada di dalamnya, termasuk teks, gambar, video, dan metadata.

  3. Mengikuti Tautan: Crawler akan mengidentifikasi semua tautan (link) yang ada di halaman tersebut, baik tautan internal (ke halaman lain di situs yang sama) maupun tautan eksternal (ke situs lain).

  4. Menambahkan ke Antrean: Tautan-tautan yang ditemukan akan ditambahkan ke dalam antrean untuk dikunjungi selanjutnya. Proses ini berulang terus-menerus, memungkinkan crawler untuk menjelajahi jaringan situs yang luas.

  5. Mengindeks dan Menyimpan: Informasi yang dikumpulkan oleh crawler kemudian diindeks dan disimpan dalam database mesin pencari. Indeks ini berfungsi sebagai “peta” bagi mesin pencari untuk menemukan dan menampilkan konten yang paling relevan saat ada permintaan pencarian. “””

Fungsi Utama Web Crawling#

Markdown cell untuk daftar fungsi#

“””

  • Indeksasi Mesin Pencari: Ini adalah fungsi yang paling umum. Web crawling memastikan bahwa konten-konten baru dan yang diperbarui dapat ditemukan dan diindeks oleh mesin pencari.

  • Analisis Data: Digunakan untuk mengumpulkan data dalam jumlah besar dari berbagai situs web untuk analisis pasar, penelitian, atau pemantauan tren.

  • Pemantauan Situs Web: Digunakan untuk memantau perubahan atau pembaruan konten di sebuah situs web. “””

Penjelasan Kode Secara Keseluruhan#

“”” Kode ini adalah skrip Python yang dirancang untuk mengambil data publikasi ilmiah dari API Springer Nature. Skrip ini bekerja dengan mengirimkan permintaan pencarian ke server API berdasarkan serangkaian kata kunci yang telah ditentukan. “””

—#

Pengaturan Awal (Inisialisasi)#

“”” Pada bagian ini, kode melakukan persiapan sebelum memulai proses pengambilan data.

  • Kunci API (api_key): Mengidentifikasi pengguna untuk mengakses layanan API Springer Nature.

  • Daftar Kata Kunci (keywords): Berisi topik-topik yang akan dicari, seperti "web mining" dan turunannya.

  • URL Endpoint API (url): Alamat server API yang akan dihubungi.

  • List Kosong (all_records): Wadah untuk menyimpan semua data publikasi yang berhasil ditemukan dari setiap kata kunci. “””

—#

Proses Pengambilan Data (Looping)#

“”” Proses ini menggunakan perulangan (for) untuk mengotomatisasi pencarian untuk setiap kata kunci dalam daftar.

  • Setiap kata kunci (kw) dari list keywords akan diproses satu per satu.

  • Sebuah permintaan HTTP GET dikirimkan ke URL API. Permintaan ini menyertakan parameter penting:

    • q: Kata kunci pencarian saat ini.

    • api_key: Kunci API Anda.

    • p: Jumlah hasil per halaman (diatur 20).

  • Fungsi requests.get() bertugas untuk mengirimkan permintaan ini ke server. “””

—#

Penanganan Respons (Validasi dan Ekstraksi)#

“”” Setelah permintaan dikirim, kode memeriksa respons dari server untuk memastikan data berhasil diambil.

  • Kode memeriksa status respons. Jika statusnya 200 (OK), berarti permintaan berhasil.

  • Data dalam format JSON diubah menjadi objek Python (dictionary) untuk diproses.

  • Selanjutnya, kode melakukan perulangan (for) untuk mengambil setiap publikasi (record) dari data hasil.

  • Informasi penting seperti DOI, judul, dan abstrak diambil. Metode .get() digunakan untuk menghindari error jika data tidak ditemukan.

  • Setiap data yang berhasil diekstrak akan dicetak ke konsol dan disimpan ke dalam list all_records sebagai sebuah dictionary. “””

—#

Penanganan Error#

“”” Bagian ini memastikan skrip tidak berhenti jika terjadi kesalahan.

  • Jika status respons dari server bukan 200 (misalnya 401 untuk otorisasi gagal atau 404 untuk halaman tidak ditemukan), kode akan mencetak pesan Error yang informatif.

  • Pesan ini mencakup kode status dan teks respons, membantu Anda mengidentifikasi penyebab kegagalan permintaan. “””

# API Key Springer Nature
api_key = "9aa2aa513b78be3c8250c64138c84ed8"

# daftar kata kunci
keywords = ["web mining", "web usage mining", "web structure mining"]

#alamat url
url = "https://api.springernature.com/meta/v2/json"

all_records = []

for kw in keywords:
    params = {
        "q": kw,
        "api_key": api_key,
        "p": 20
    }

    response = requests.get(url, params=params)

    if response.status_code == 200:
        data = response.json()
        print(f"Kata kunci: {kw} | Total hasil: {data['result'][0]['total']}\n")

        for record in data['records']:
            doi = record.get('doi', 'N/A')
            title = record.get('title', 'No title')
            abstract = record.get('abstract', 'No abstract')

            print(f"DOI: {doi}")
            print(f"Title: {title}")
            print(f"Abstract: {abstract}\n")

            all_records.append({
                "Keyword": kw,
                "DOI": doi,
                "Title": title,
                "Abstract": abstract
            })
    else:
        print("Error:", response.status_code, response.text)

Proses Penyimpanan#

“””

  1. Mengubah List Menjadi DataFrame:

    • print("✅ Semua hasil sudah disimpan ke zaincsv.csv"): Setelah proses selesai, pesan konfirmasi ini dicetak untuk memberitahu Anda bahwa data telah berhasil disimpan. “””
